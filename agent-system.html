🎭 오케스트라가 히스토리를 참조하는 방법
  • 새 업무 요청 시 work-history.json에서 유사 업무 검색
  • 이전에 사용된 에이전트 순서 참고
  • 실패·주의사항 notes 확인 후 같은 실수 방지
  • 의료 관련 변경 시 이전 임상 근거 references 참조
  • 파일 변경 이력으로 충돌 가능성 예측
📝 히스토리 기록 시점
  • 배포 에이전트가 배포 완료 직후 자동 기록
  • QA 에이전트가 검증 결과 포함
  • 각 팀 에이전트가 사용한 리뷰어 에이전트 목록 포함
  • 의료 변경의 경우 근거 문헌 링크 필수 포함
  • 실패 배포도 기록 (원인 + 해결 방법)
